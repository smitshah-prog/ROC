# Testing Strategy

## 🧪 Testing Types

### 1. Unit Testing

* Test backend logic
* Tool: Jest

---

### 2. API Testing

* Validate endpoints
* Tool: Supertest

---

### 3. UI Testing

* React Testing Library

---

### 4. Multi-User Testing (Important)

* Test simultaneous users
* Check data conflicts

---

## 📌 Key Scenarios

* Multiple users editing same record
* Role-based access validation
* Data consistency

---

## 🔁 CI/CD Testing

* Run tests before deployment

---

## 📌 Performance Testing

* Test with 5–20 users on LAN

---

## 📌 QA Checklist

* All modules tested
* No data loss
* No crashes
